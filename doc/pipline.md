阶段1：打牢基础（先会“做”，再会“讲”）
目标：能独立写出可运行的单 Agent 应用，并解释每一层做了什么。

必学：
Prompt 设计与系统提示词
ChatModel 调用、参数（温度、max tokens、超时、重试）
Tools（函数签名、参数校验、错误处理）
create_agent 执行流（模型决策 -> 调工具 -> 回答）
.env、配置管理、日志
产出项目（你已经有雏形）：
CLI 智能体：支持工具调用（计算器、时间、HTTP 查询）
面试表达要点：
“智能体和普通 Chat Completion 的区别是什么？”
“为什么要把工具设计为可观测、可重试的模块？”
阶段2：智能体进阶（岗位最看重）
目标：从“会调用工具”升级到“会构建多步骤任务代理”。

必学：
记忆：短期会话记忆 vs 长期记忆（存储）
RAG：检索、重排、上下文压缩
多工具编排：搜索、数据库、代码执行（受限沙箱）
多智能体协作：Router / Planner / Worker 模式
人类反馈与中断（human-in-the-loop）
产出项目（建议做 2 个）：
文档问答 Agent（RAG）
任务执行 Agent（可调用外部 API + 失败恢复）
面试表达要点：
“如何避免工具调用死循环？”
“如何在多轮对话里控制 token 成本和时延？”
阶段3：工程化与上线能力（区分初中高级）
目标：让项目具备“可上线、可维护、可评估”的能力。

必学：
评估体系：准确率、任务成功率、工具调用成功率、时延、成本
观测与追踪（LangSmith 思路即可）
Guardrails：敏感词、越权工具调用、Prompt Injection 防护
缓存、并发、降级、熔断
API 化部署（FastAPI）与容器化（Docker）
产出项目（建议 1 个综合项目）：
可部署 Agent 服务（带日志、评估脚本、基础监控）
面试表达要点：
“如何做 Agent 的线上故障排查？”
“你如何定义并持续优化智能体 KPI？”
阶段4：求职冲刺（拿面试机会）
目标：把“会做项目”转成“可被面试官感知的竞争力”。

重点：
业务目标（解决了什么问题）
技术方案（模型、工具链、架构）
指标结果（成功率/时延/成本优化）
你的贡献（不是“参与”，而是“主导了什么”）
准备：
手撕/口述：Agent 执行链路、RAG 流程、异常处理
系统设计：设计一个客服/投研/运营自动化 Agent 平台
行为：如何处理幻觉、线上事故、需求变更
建议：
GitHub 至少 3 个可运行项目
每个项目有 README、架构图、评估结果、演示视频/GIF
每周学习节奏（可长期复用）
5 天学习建议：
2 天：概念 + 小实验
2 天：项目编码
1 天：复盘 + 文档 + 面试题
每周固定产出：
1 个可运行迭代
1 篇技术笔记

在当前项目上做 3 个升级：
增加一个真实工具（如网页检索或本地知识库检索）
增加会话记忆（最近 N 轮）
增加日志与错误处理（工具失败时给出可解释回复）

你的 Agent 架构图和执行流程
如何解决稳定性问题
如何评估“这个 Agent 是否真的有用”
